Freeing Up Space on the /boot File SystemThe Original Poster has a separate /boot partition, and that is what is full and preventing the apt system from working. It will be necessary for him to free up space there.
If there almost enough space, go to /boot and remove a config file or two:
ubuntu_root@ubuntu-services:/boot$ sudo rm vmlinuz-4.2.0-35-generic
ubuntu_root@ubuntu-services:/boot$ sudo rm System.map-4.2.0-35-generic
ubuntu_root@ubuntu-services:/boot$ sudo rm initrd.img-4.2.0-35-generic
ubuntu_root@ubuntu-services:/boot$ sudo rm config-4.2.0-35-generic
ubuntu_root@ubuntu-services:/boot$ sudo rm abi-4.2.0-35-generic
ubuntu_root@ubuntu-services:/boot$ df -h
